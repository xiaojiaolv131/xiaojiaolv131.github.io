    • 百度面试记录

    第一次去百度面试，还是紧张的。这家公司是以前很想加入的公司，现在貌似没那么渴望了，但是作为一次面试经历，还是记录一下好了。

    一面

    一面是个戴眼镜的女生，不知该称呼小姐姐还是小妹妹，反正聊得很好，关于技术层面主要聊了这些：

    • 一贯的自我介绍
    • 谈谈vue.js双向数据绑定的认识，他们有自己写的一套东西，我顺便了解了一下
    • 说说Vuex
    • 你了解underscore吗？
    • 如何得到一个DOM结构中所有不同标签出现的数量
    • 如果需要实现一个目录结构，要怎样来实现呢？ （这个早在三年前就经常写这个了）
    • 知道浏览器缓存吗？
    • ES6用的多吗？
    • 其他的忘了。。。。

    二面

    二面是一大哥，喜欢说话闭眼睛的大哥，过来时拿了一打儿A4大白纸和一支黑色签字笔，还有他的mac破肉，被一打儿大白纸吓尿了，以为要手写各种算法、处理函数之类的，二面上来就询问我一面都问了啥，我简单说了一下之后，其他问题：哪个大学的、哪年毕业的、一直在这家公司吗？然后，貌似真正的深层技术面开始了：

    • 写一个类
    • 让我修改这个类，已达到他想要的调用方式，比如：链式调用
    • 说下vue-router
    • 说说Ajax实现机制（怎么区分返回时哪个请求），有哪些状态值
    • 说说跨域，我提到了iframe、jsonp、cors，说说怎么实现的
    • 说说http／https协议
    • 说说浏览器缓存，我讲了强缓存和协商缓存，他继续提问：怎样越过强缓存
    • 说说数据缓存，我只提到了cookies，sessionStorage 和 localStorage，区别是啥？另外，他还提了一句session（他说我说了，其实我没说）
    • 说说JQuery选择器怎样实现的，我按照链式路径的解析来说的，然后，说说JQuery有哪些选择器？这尼玛好久不用了，会用都叫不上名字了，然后面试官引导性的说原生有几种获取DOM节点的方法，我列举了常用的，其中提到了document.querySelector，然后他说这不就是选择器嘛。。。
    • 说说git版本控制，比如有一些commit，但未push的提交，如何只合并最近提交的前5个。平时都是sourcetree嘛，那个时间近就提交哪个嘛，反正很少用命令去搞
    • ES6规范经常用吗？用的不多诶
    • 了解require.js模块化工具吗？模块化标准都有哪些？有啥区别？我只说了AMD和CMD，区别忘了，反正是异步的
    • 说说Promise，怎么实现的
    • 说说webpack，如果有一个入口index.js，其中引用了a.js和b.js，那么打出来的包长啥样？常用的loader有哪些？
    • 说说安全方面的东西
    • 咱们几点开始的？我也忘了。。。。
    • 有些问题这大哥应该是想挖坑的，但我才疏学浅，估计也没了挖坑的热情了

    以上这些问题，有很多都是止步于表面了解，即便之前写过相关文章，也忘的一干二净了，对于一个前端来讲，这些在基础不过了，因此，还需要多看，但有的东西，只是看，并没有实践过，因此，很难记忆深刻，且失去了变通。
